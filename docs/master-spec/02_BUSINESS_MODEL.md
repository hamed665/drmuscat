# DrMuscat Business Model — V10.1

## 1. Main Monetization Logic
DrMuscat is free for public users. Revenue comes from healthcare/wellness providers and premium visibility products.

Primary monetization:
1. provider subscription plans for centers and doctors
2. verified profile upgrades
3. featured/partner placements
4. internal advertising and sponsored placements
5. CPC campaigns funded by provider wallet
6. media/video packages
7. SEO/content/profile enhancement packages
8. microsites or premium landing pages if approved
9. manual onboarding and partner packages

Deprecated and forbidden in MVP:
- user paid card sales
- Health Card membership sales
- required paid user access
- selling a card as the core product

## 2. Provider Plan Ladder
Centers and doctors may register under four plan levels. The first plan is free; paid plans add trust, visibility, analytics and growth tools. Paid status must never imply medical superiority.

| Feature | Free Listing | Verified Starter | Growth Partner | Premium / Ads Pro |
|---|---:|---:|---:|---:|
| Public profile | Yes | Yes | Yes | Yes |
| Claim profile | Yes | Yes | Yes | Yes |
| Verification review | No / basic | Yes | Yes | Yes |
| Verified badge | No | Yes | Yes | Yes |
| Logo/cover/gallery | Limited | Standard | Expanded | Premium |
| Services/branches/doctors | Basic | Standard | Expanded | Expanded |
| Public offers | No or limited | Limited | Yes | Yes |
| WhatsApp/call/direction tracking | Basic | Basic | Advanced | Advanced |
| Analytics | No/basic | Basic | Advanced | Advanced + reports |
| Review response | No | Yes | Yes | Yes |
| Featured placement | No | Add-on | Limited/add-on | Included/priority |
| Ads wallet/CPC | No | Add-on/future | Yes | Yes |
| Monthly visibility report | No | Basic | Yes | Yes |
| Sales support | No | Basic | Yes | Priority |

## 3. Free Plan Rules
Free listings are allowed so the site is not empty and can build SEO. Free profiles can be unclaimed or claimed, but they must not display partner/verified/premium language unless verification is approved.

Free listings may include:
- public name
- category
- area
- phone/WhatsApp if publicly available or submitted
- directions
- basic services
- unclaimed profile label
- claim/suggest edit/report wrong info CTAs

## 4. User Access Rules
Users register for free. Browsing must not be blocked behind login. Public users can view centers, doctors, categories, areas, service pages, public offers, reviews that meet moderation thresholds, contact buttons and directions without registration.

Soft registration nudges are required:
- sticky or contextual “Create free account” prompt
- after repeated profile views
- when saving a center
- when claiming an offer code
- when writing a review
- when requesting reminders or alerts

Do not use aggressive popups that damage SEO, mobile UX or trust.

## 5. Manual Billing First
MVP uses manual billing:
- invoice generated by system
- receipt uploaded
- admin verifies
- subscription activated
- ledger entries created
- commission calculated if referral exists

Do not implement automatic payment gateway in Phase 1 unless explicitly approved.

## 6. Marketer Commission Model
Marketer role is approved manually. Marketer receives referral code. When a referred business pays, backend calculates commission using `commission_rules` and ledger RPC. Marketer cannot mark payments as paid. Marketer cannot self-approve. Commission payouts require admin/finance approval.

## 7. Retention Logic
Centers renew if dashboard proves value:
- profile views
- WhatsApp clicks
- call clicks
- direction clicks
- leads
- offer claims/redemptions
- media/video views
- ad performance
- review trends
- profile completeness

No fake analytics. Empty states must explain what to improve.

## 8. Internal Advertising Revenue
DrMuscat must support paid visibility as a separate monetization layer in addition to subscriptions. Advertising products must include flat sponsored placements, CPC campaigns, optional future CPM, wallet-funded campaigns, sponsored category/area/service placements, and homepage/banner sponsorships.

Advertising must never replace trust. Sponsored content must be clearly labeled, medically compliant, admin-approved, measurable, and reported to centers. Pricing must be configurable by placement from admin, because homepage hero placement, category sponsorship, service-page sponsorship, search promoted card, and offer highlight placements have different commercial value.

Advertising revenue must connect to center wallets, ledger/audit logs, campaign reports, and monthly visibility reports.
