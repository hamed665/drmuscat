# Product Scope and MVP Boundaries — V10.1

## 1. Launch MVP Must Include
- English and Arabic public site
- SEO-first public architecture
- homepage
- centers directory
- center profile pages
- doctors directory/profile architecture
- category, area, service and category×area SEO pages
- claim profile flow
- suggest edit/report wrong information
- provider four-plan registration model
- free user registration
- anonymous browsing without login wall
- soft registration nudges
- WhatsApp/call/direction click tracking
- basic admin for listings, claims and SEO controls
- public offers architecture without paid user card
- sitemap, robots, canonical, hreflang, metadata and JSON-LD

## 2. Explicitly Cancelled From MVP
- Health Card sales
- paid user card memberships
- any user payment requirement to browse centers/doctors/offers
- Persian/Farsi public routes or SEO
- Hindi public SEO routes in launch

## 3. Deferred Features
- AI chat full RAG
- ads wallet/CPC self-service
- advanced reviews
- monthly PDF reports
- advanced CRM
- full center dashboard
- full doctor dashboard
- media pipeline automation

Deferred features must have clean extension points but must not delay the first launch.

## 4. Medical and Trust Boundaries
- no diagnosis
- no prescription
- no emergency triage as a substitute for licensed care
- no fake reviews or ratings
- no fake verified/partner status
- no medical outcome guarantees
- no paid ranking presented as organic ranking

## 5. Public Access Boundary
Public users can view all public discovery pages without login. Login prompts are allowed only as non-blocking nudges unless the action requires identity, anti-spam, personalization, review integrity, or offer-code tracking.
