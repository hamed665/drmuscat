# DrMuscat V10.3 — Competitor Parity and Gap Matrix

## 1. Purpose
This file translates global healthcare-platform patterns into DrMuscat requirements without blindly copying expensive features too early.

## 2. Global Platform Patterns
Successful healthcare platforms commonly include:
- Search and discovery.
- Insurance filtering.
- Verified provider profiles.
- Reviews and trust systems.
- Appointment request or booking.
- Provider dashboards.
- Analytics and performance reports.
- Sponsored visibility with clear labels.
- Content/SEO authority.
- Mobile-first UX.

## 3. DrMuscat Launch Priority
For Oman launch, prioritize:
1. SEO discovery.
2. Trustworthy center/doctor profiles.
3. WhatsApp-first conversion.
4. Claim/profile management.
5. Admin operational control.
6. Data quality/import/duplicate tools.
7. Manual monetization.
8. Sponsored visibility.
9. Appointment request capture.
10. Insurance acceptance.

## 4. Deferred Global Features
Do not build in MVP unless approved:
- Full live appointment marketplace.
- Real-time insurance eligibility.
- Medicine delivery.
- Lab booking marketplace.
- Patient medical records.
- Video consultation infrastructure.
- Mobile native app.
- AI diagnosis/triage.
