# DrMuscat Master Spec V10.4 — Agent-Safe Build Framework Send Order

## What This Package Is
This package is the canonical DrMuscat Master Spec V10.4. It includes V10.3.1 operational/admin/SEO hardening and adds an Agent-Safe Build Framework so Codex, Claude Code, or any coding agent can build from zero with phase locks, stop rules, validation gates, migration discipline, SEO route contracts, RLS/security controls, and human approval checkpoints.

The goal is not to make the product larger. The goal is to make implementation safer, more testable, and less dependent on agent guessing. A rare outbreak of sanity, suspicious but useful.

## Mandatory Execution Rule
Agents must start in `SAFE_REVIEW_ONLY` or `PHASED_BUILD_ONLY`. They must not build the full platform in one pass. Start with Phase 0 only, produce the readiness report, then stop for approval.

## Non-Negotiable Build Rules
- English and Arabic only at launch.
- Oman only at launch.
- Country-aware routes only: `/[locale]/[country]/...`.
- No Persian/Hindi public SEO routes.
- No Health Card/card sales. Patient offers replace cards.
- Anonymous browsing must work on public pages.
- User registration is free.
- `geo_areas` is canonical; legacy `areas` is not a writable source of truth.
- `doctor_practice_locations` is canonical; legacy `doctor_centers` is not a writable source of truth.
- `provider_plans` and `platform_settings` are canonical.
- Sponsored slots must be clearly labeled and must never secretly alter organic ranking.
- No live payment gateway, AI chat, Meilisearch, Web Push, or multi-country launch in MVP unless explicitly approved in a later phase.

## Required Start Prompt
Use `77_FINAL_BUILD_FROM_ZERO_PROMPT.md` when starting a clean repository. The agent must begin with Phase 0 only.
